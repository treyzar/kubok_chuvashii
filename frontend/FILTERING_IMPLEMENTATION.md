# Реализация фильтрации и поиска

## Что реализовано

### 1. Инфраструктура локализации ✅
- **Файл**: `src/lib/localization.ts`
- Все строки интерфейса на русском языке
- Функции для получения локализованных меток статусов
- Обработка ошибок с русскими сообщениями

### 2. Утилиты для работы с датами ✅
- **Файл**: `src/lib/dateUtils.ts`
- Форматирование дат в русской локали (DD.MM.YYYY)
- Форматирование даты и времени (DD.MM.YYYY HH:MM)
- Русские названия месяцев
- Валидация диапазонов дат
- Относительное время ("2 дня назад")

### 3. Утилиты для фильтров ✅
- **Файл**: `src/lib/filterUtils.ts`
- Проверка активности фильтров
- Сериализация/десериализация параметров URL
- Валидация значений фильтров
- Санитизация входных данных

### 4. Хук debounce ✅
- **Файл**: `src/hooks/useDebounce.ts`
- Задержка 300мс для поиска
- Предотвращение избыточных API запросов

### 5. Страница списка обращений с фильтрами ✅
- **Файл**: `src/pages/crm/AppealsList.tsx`

#### Реализованные фильтры:
1. **Поиск** - семантический поиск через API (параметр `query`)
   - Debounce 300мс
   - Кнопка очистки поиска
   - Поиск по описанию обращений

2. **Фильтр по статусу** - фильтрация по статусу обращения
   - Все статусы / Новое / В работе / Решено / Отклонено
   - Параметр API: `status_id`

3. **Фильтр по категории** - выбор категории обращения
   - Загрузка категорий из API `/public/categories`
   - Динамическое обновление подкатегорий

4. **Фильтр по подкатегории** - выбор подкатегории
   - Отображается только при выборе категории
   - Автоматический сброс при смене категории
   - Параметр API: `subcategory_id`

5. **Фильтр по тегам** ✅ - множественный выбор тегов
   - Динамическая загрузка доступных тегов из обращений
   - Клик по тегу для добавления/удаления из фильтра
   - Визуальная индикация выбранных тегов (иконка X)
   - Цветовая кодировка тегов:
     - 🔴 Красный - срочные теги
     - 🟡 Желтый - требует проверки
     - 🔵 Синий - передано в департамент
     - ⚫ Серый - дубликаты
     - 🟣 Фиолетовый - прочие теги
   - Клиентская фильтрация (AND логика - все выбранные теги должны присутствовать)
   - Теги кликабельны как в фильтре, так и на карточках обращений

6. **Обзор по категориям и подкатегориям** ✅ - визуальная панель навигации
   - Сворачиваемая панель с категориями
   - Счетчики обращений для каждой категории
   - Клик по категории для фильтрации
   - Автоматическое раскрытие подкатегорий при выборе категории
   - Счетчики обращений для каждой подкатегории
   - Клик по подкатегории для детальной фильтрации
   - Визуальная индикация выбранных категорий/подкатегорий
   - Анимированные переходы при раскрытии/сворачивании

#### Дополнительные возможности:
- **Синхронизация с URL** - все фильтры сохраняются в URL параметрах
- **Кнопка "Очистить фильтры"** - сброс всех фильтров одним кликом
- **Счетчик результатов** - отображение количества найденных обращений
- **Состояние загрузки** - индикатор загрузки и блокировка фильтров
- **Пустые результаты** - разные сообщения для пустого списка и отсутствия результатов

## Использование API

### Параметры запроса `/tickets`:
```typescript
{
  query?: string,           // Семантический поиск
  status_id?: string,       // init | open | closed | rejected
  subcategory_id?: number,  // ID подкатегории
  limit?: number,           // Лимит результатов (по умолчанию 50)
  offset?: number           // Смещение для пагинации
}
```

### Пример запроса:
```
GET /api/v1/tickets?query=отопление&status_id=open&subcategory_id=1&limit=50
```

### Фильтрация по тегам:
Фильтрация по тегам выполняется на клиенте после получения данных из API, так как API возвращает теги в полях `tag_ids` и `tag_names` для каждого обращения.

## Как использовать

1. **Поиск**: Введите текст в поле поиска - результаты обновятся через 300мс
2. **Фильтр по статусу**: Выберите статус из выпадающего списка
3. **Фильтр по категории**: Выберите категорию - появится фильтр подкатегорий
4. **Фильтр по подкатегории**: Выберите подкатегорию для более точной фильтрации
5. **Фильтр по тегам**: 
   - Кликните на тег в секции "Фильтры по тегам" для добавления в фильтр
   - Кликните повторно для удаления из фильтра
   - Можно выбрать несколько тегов (показываются только обращения со ВСЕМИ выбранными тегами)
   - Теги также кликабельны на карточках обращений
6. **Обзор по категориям**:
   - Кликните на заголовок "Обзор по категориям" для сворачивания/разворачивания панели
   - Кликните на категорию для просмотра обращений этой категории
   - При выборе категории автоматически раскроются её подкатегории со счетчиками
   - Кликните на подкатегорию для более детальной фильтрации
   - Счетчики показывают количество обращений в каждой категории/подкатегории
   - Кликните на выбранную категорию повторно для сброса фильтра
7. **Очистка**: Нажмите кнопку "Очистить фильтры" для сброса всех фильтров

## URL параметры

Фильтры сохраняются в URL, что позволяет:
- Делиться ссылками с примененными фильтрами
- Использовать кнопки "Назад"/"Вперед" браузера
- Обновлять страницу без потери фильтров

Пример URL:
```
/crm/appeals?search=отопление&status=open&category=1&subcategory=2&tags=Срочно,Проверка
```

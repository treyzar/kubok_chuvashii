# ⚡ Быстрый запуск за 5 минут

## 🎯 Что вы получите

После выполнения этих шагов у вас будет:
- ✅ Работающая CRM-система для обращений граждан
- ✅ AI-детекция дубликатов обращений
- ✅ Публичная форма для подачи обращений
- ✅ Тепловая карта проблемных зон
- ✅ Аналитика и статистика в реальном времени

## 📋 Требования

- **Go** 1.25.5+
- **Node.js** 18+
- **Docker** и Docker Compose
- **Make**

## 🚀 Шаги запуска

### 1. Запустите базы данных

```bash
cd TicketService
make databases
sleep 3
```

Это запустит PostgreSQL и Redis в Docker контейнерах.

### 2. Примените миграции

```bash
make migrate
```

Создаст все необходимые таблицы и сгенерирует Go код.

### 3. Загрузите тестовые данные

```bash
make mock
```

Создаст:
- 30+ тестовых обращений с реальными AI-эмбеддингами
- 6 групп дубликатов для тестирования
- 4 тестовых пользователя
- Категории, департаменты, теги

**Примечание**: Требуется Ollama с моделью `nomic-embed-text-v2-moe`. Если недоступен, будут использованы fallback-эмбеддинги.

### 4. Запустите backend

```bash
make serve
```

API будет доступен на http://localhost:8888

### 5. Запустите frontend

В новом терминале:

```bash
cd ../frontend
npm install
npm run dev
```

Frontend будет доступен на http://localhost:3000

## 🧪 Что попробовать

### 1. Подать обращение через публичную форму

1. Откройте http://localhost:3000
2. Заполните форму:
   - Категория: ЖКХ → Отопление
   - Адрес: ул. Ленина, 45
   - Описание: Нет отопления в доме
   - ФИО: Иванов Иван Иванович
   - Телефон: +7 900 123 45 67
3. Нажмите "Отправить обращение"

### 2. Войти в CRM и посмотреть обращения

1. Откройте http://localhost:3000/crm/login
2. Войдите как **admin@crm.local** / **admin123**
3. Увидите:
   - Dashboard с статистикой
   - Список всех обращений
   - Тепловую карту

### 3. Протестировать AI-детекцию дубликатов

1. В CRM откройте любое обращение про отопление
2. Справа увидите блок "Возможные дубликаты"
3. Система покажет похожие обращения с процентом схожести
4. Можно объединить дубликаты одной кнопкой

### 4. Проверить разграничение доступа

1. Выйдите из системы
2. Войдите как **gkh@crm.local** / **gkh123**
3. Увидите ТОЛЬКО обращения департамента ЖКХ
4. Статистика тоже будет только по ЖКХ

### 5. Создать нового пользователя

1. Войдите как администратор
2. Перейдите в "Админ-панель"
3. Нажмите "Создать пользователя"
4. Заполните данные и выберите департамент
5. Новый пользователь сможет войти в систему

## 🔧 Полезные команды

```bash
# Backend
cd TicketService
make serve              # Запустить API
make databases          # Запустить БД
make databases-down     # Остановить БД
make migrate            # Применить миграции
make mock               # Загрузить тестовые данные
make postgres           # Подключиться к PostgreSQL
make redis              # Подключиться к Redis

# Frontend
cd frontend
npm run dev             # Запустить dev сервер
npm run build           # Собрать для продакшена
npm run lint            # Проверить код
```

## ❓ Проблемы?

### Backend не запускается

```bash
cd TicketService
go mod tidy
make serve
```

### Ошибка подключения к БД

```bash
make databases-down
make databases
sleep 3
make migrate
```

### Frontend не запускается

```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### Не видны обращения после входа

Проверьте:
1. Запущен ли backend (`make serve`)
2. Правильный ли email/пароль
3. Есть ли тестовые данные (`make mock`)

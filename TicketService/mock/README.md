# Mock Data Generator

Этот скрипт генерирует реалистичные тестовые данные для системы обращений граждан.

## Что создается

### Справочники
- 8 категорий обращений (ЖКХ, Дороги, Благоустройство и т.д.)
- 32 подкатегории
- 8 подразделений
- 7 внутренних тегов
- 6 источников обращений

### Пользователи
- `admin@crm.local` / `admin123` - Администратор
- `gkh@crm.local` / `gkh123` - Сотрудник ЖКХ
- `transport@crm.local` / `transport123` - Сотрудник дорожного отдела
- `ecology@crm.local` / `ecology123` - Сотрудник экологического отдела

### Обращения (30 штук)

#### Группы дубликатов для тестирования AI:

1. **Проблемы с отоплением** (3 дубликата)
   - Дом на улице Ленина 45, нет отопления
   - Разные формулировки одной проблемы
   - Статусы: Open, Open, Init

2. **Ямы на дорогах** (3 дубликата)
   - Перекресток Гагарина и Мира
   - Яма глубиной 25-30 см
   - Статусы: Open, Open, Init

3. **Проблемы с освещением** (2 дубликата)
   - Улица Победы, не работают фонари
   - Статусы: Open, Open

4. **Мусор во дворах** (2 дубликата)
   - Дом 12 на улице Калинина
   - Статусы: Open, Init

5. **Детские площадки** (2 дубликата)
   - Дом 8 на улице Чапаева
   - Статусы: Open, Init

6. **Бродячие собаки** (2 дубликата)
   - Дом 7 на улице Комсомольской
   - Статусы: Open, Init

#### Уникальные обращения (15 штук)
- Ржавая вода
- Неубранный снег
- Очереди в поликлинике
- Протечка крыши в школе
- Плохой интернет
- Несанкционированная свалка
- Высокие счета за ЖКУ
- Озеленение двора
- Задержки автобусов
- Нехватка мест в детском саду

#### Закрытые обращения (3 штуки)
- Для статистики и истории

## Как запустить

```bash
cd TicketService
go run mock/main.go mock/tickets_data.go
```

## Что происходит

1. Очищаются все существующие данные
2. Создаются справочники (категории, подразделения, теги)
3. Создаются пользователи с хешированными паролями
4. Создаются обращения с AI embeddings для поиска дубликатов
5. Добавляются комментарии к обращениям
6. Назначаются теги

## Тестирование дубликатов

После генерации данных:

1. Войдите в CRM под любым пользователем
2. Откройте любое обращение из группы дубликатов
3. Система автоматически покажет карточку "Система ИИ обнаружила возможные дубликаты"
4. Вы увидите похожие обращения с процентом совпадения (70%+)
5. Можно нажать "Объединить" для слияния дубликатов

## Особенности

- Все обращения имеют реалистичные описания
- Координаты соответствуют городу Чебоксары
- Даты создания распределены от 45 дней назад до текущего момента
- AI embeddings генерируются для каждого обращения
- Дубликаты имеют схожие описания, но разные формулировки
- Порог схожести для определения дубликатов: 70%

## Примечания

- Генерация embeddings может занять некоторое время
- Требуется подключение к OpenAI API для embeddings
- Если API недоступен, используются fallback embeddings
